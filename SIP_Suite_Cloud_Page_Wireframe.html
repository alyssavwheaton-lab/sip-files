Cloud Page Build Notes

Technical and content notes for Salesforce Cloud Pages implementation

Video Embeds

All videos hosted on Kaltura (RJ MediaSpace). Use secure iframe embeds with referrerPolicy="no-referrer-when-downgrade". Three video assets currently live:

1_a5fz5af5 Camp SIP Suite Overview (hero)
1_111xazco January Playbook (Brad)
1_0zvaecal December Playbook (James)

Monthly Update Cadence

New playbook videos publish every third Thursday. Investment team meets second Thursday evening, email script drafted Wed/Thu, video goes live the following Thursday. Cloud page should surface newest episode with LATEST badge. Previous months shift down. Consider slider component for 4+ episodes.

Factsheet PDFs

Host current Q2 2025 factsheets as downloadable PDFs. Update quarterly. Three assets: SIP, SIP+, GSIP. Match filenames to 2Q25_[Product]_Factsheet_RJ.pdf convention. Consider inline viewer for mobile.

Migration Path

This cloud page serves as interim hub while RJnet page is built with Earl. Once RJnet page is live, cloud page URL should 301 redirect. Seismic DSR links should point to cloud page initially, then update to RJnet. All email campaign URLs should use UTM parameters for tracking.

Webinar Integration

Registration CTAs should link to On24 registration page. Post-webinar, swap CTA to "Watch Replay" and embed recording. Plan for quarterly rotation: 2/26 Income Webinar, 3/18 Meet the Managers. Add dynamic content block if SF supports conditional display by date.

Tracking & Analytics

Implement UTM structure: ?utm_source=email&utm_medium=cloud_page&utm_campaign=sip_suite_2026. Track video play events via Kaltura analytics. Track PDF downloads. Pass engagement data back to SF for lead scoring and segmentation for Email 2 (SIP+) and Email 3 (GSIP) sends.
